Firebase Setup Cheat Sheet

One-page reference for new project setup. Print and tick.

One-time project setup Cloud

  • Console Create project at console.firebase.google.com
  • Console Add Android app → enter package name → download google-services.json → place in android/app/
  • Console Authentication → Sign-in method → Google → Enable
  • Console Firestore → Create database → choose region
  • Console Firestore → Rules → paste security rules (see below)
  • Console Project settings → Your apps → Add SHA-1 fingerprint
  • CLI flutterfire configure → generates lib/firebase_options.dart

Get SHA-1 (debug key):

keytool -list -v \
  -keystore ~/.android/debug.keystore \
  -alias androiddebugkey \
  -storepass android -keypass android

Flutter project setup Code

pubspec.yaml

dependencies:
  firebase_core: ^3.0.0
  firebase_auth: ^5.0.0
  cloud_firestore: ^5.0.0
  google_sign_in: ^6.1.0

android/settings.gradle.kts — declare plugin:

id("com.google.gms.google-services") version "4.4.2" apply false

android/app/build.gradle.kts — apply plugin:

id("com.google.gms.google-services")

lib/main.dart — always first:

WidgetsFlutterBinding.ensureInitialized();
await Firebase.initializeApp(
  options: DefaultFirebaseOptions.currentPlatform,
);

Auth pattern Code

// Sign in (Google)
final googleUser = await GoogleSignIn().signIn();
final googleAuth = await googleUser!.authentication;
final credential = GoogleAuthProvider.credential(
  accessToken: googleAuth.accessToken,
  idToken: googleAuth.idToken,
);
await FirebaseAuth.instance.signInWithCredential(credential);

// Current user
final user = FirebaseAuth.instance.currentUser; // null if not logged in

// Listen to auth changes
FirebaseAuth.instance.authStateChanges().listen((user) { ... });

// Sign out
await FirebaseAuth.instance.signOut();

Firestore patterns Code

final db = FirebaseFirestore.instance;

// Write (overwrites entire document)
await db.collection('users').doc(uid).set({...});

// Merge (partial update — other fields survive)
await db.collection('users').doc(uid).set(
  {'lastSeen': Timestamp.now()},
  SetOptions(merge: true),
);

// Read once
final doc = await db.collection('users').doc(uid).get();
final data = doc.data(); // Map<String, dynamic>? — null if missing

// Read subcollection
final snap = await db
    .collection('users').doc(uid)
    .collection('months').get();
for (final d in snap.docs) { print(d.id, d.data()); }

// Real-time listener
db.collection('users').doc(uid).snapshots().listen((snap) {
  print(snap.data()); // fires on every server change
});

Security rules Console

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Each user can only access their own subtree
    match /users/{uid}/{document=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == uid;
    }
  }
}

Key concepts

TermWhat it is
google-services.jsonBuild-time Android config. Downloaded from Firebase console. Gradle reads it.
firebase_options.dartRuntime Flutter config. Generated by flutterfire configure. SDK reads it.
SHA-1 fingerprintHash of your APK's signing certificate. Firebase uses it to verify your app's identity for Google Sign-In.
Google Services pluginGradle plugin that processes google-services.json and injects Firebase IDs into the APK.
user.uidFirebase's unique ID for the logged-in user. Stable across sessions. Use as Firestore document ID.
